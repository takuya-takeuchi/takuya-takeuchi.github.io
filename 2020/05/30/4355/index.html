    Xamarinで実機デバッグが出来なくなった場合のエラー

    Introduction

    以前は動いていたソースも変えていないiOSアプリを実機でバッグなり焦った際のメモ。
    こんなエラーがでます。

    • MT1001: Could not find an application at the specified directory:
    • MT1006: Could not install the application

    Could not install the applicationっていうから、書き込み先に権限が無いのか？とか思っていてMac側を調べても特におかしいことは無し。
    ただ、書き込み先のバイナリが古いのが気になりました。
    古いのは書き込めなかったからだと思いますが。

    How to resolve?

    解決策がQiitaにありました。

    Qiita
    Xamarinで作成したiOSアプリが実機デバッグ起動できなくなった場合の対処方法 - Qiita
    https://qiita.com/gnk263/items/e43796e96249f03fb70b

    要する実機アイコンから一度アプリを削除しろということです。
    当然、データも消えますし、一度プロファイルでインストールしたアプリの承認も行う必要があります。

    面倒ですが､とりあえず解決。
