    が、仮想環境の使い方としてはよろしくないので、正しく使う方法を。 前提として、Python 3.3以上が対象です。

    Resolution

    仮想環境の作成

    まず、仮想環境を管理するディレクトリ(以下管理ディレクトリ)を用意します。どこでも良いです。 1カ所で管理するほうが綺麗です。 [code lang=”dos”] $ cd <管理ディレクトリ> $ python -m venv <仮想環境名> [/code] 正確には、 [code lang=”dos”] $ python -m venv <仮想環境の作成先ディレクトのパス> [/code] になります。

    仮想環境の実行

    Windowsの場合

    [code lang=”dos”] $ cd <仮想環境の作成先ディレクトのパス> $ Scripts\activate.bat [/code]

    Linuxの場合

    [code lang=”shell”] $ cd <仮想環境の作成先ディレクトのパス> $ source bin/activate [/code]

    パッケージの確認

    pipが最初から入っているはずです。 [code lang=”dos”] $ python -m pip freeze [/code] なにも表示されないです。きちんとパッケージがほかの環境と独立していることがわかります。

    仮想環境の削除

    仮想環境のフォルダを削除するだけです。
