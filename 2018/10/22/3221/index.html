    • 開発メモ その138 ChainerでDCGANのサンプル実行中に遭遇したエラー

    Introduction

    chainerに付属している**DCGAN(Deep Convolution Generative Adversarial Networks)**のサンプルを動かした際のエラーについての対応メモ。

    Resolution

    backend がない

    下記のエラーが出る。 [code lang=”cmd”] >python -i train_dcgan.py Traceback (most recent call last): File “train_dcgan.py”, line 9, in from net import Discriminator File “D:\Works\Python\Envs\chainer\chainer\examples\dcgan\net.py”, line 5, in from chainer import backend ImportError: cannot import name ‘backend’ [/code] 理由としては、現時点(2018/10/21)でのgithubのリポジトリがChainerのv5.0へ移行し始めている。 その過程でI/Fに下記の変更がはいった模様。

    要するに、 [code lang=”python”] from chainer import backends [/code] が [code lang=”python”] from chainer import backend [/code] こうなったと。 で、pipからインストールしたchainerは [code lang=”cmd”] python -m pip freeze chainer certifi==2018.10.15 chainer==4.5.0 colorama==0.3.9 cupy==4.5.0 fastrlock==0.4 filelock==3.0.9 isort==4.3.4 lazy-object-proxy==1.3.1 mccabe==0.6.1 numpy==1.15.2 protobuf==3.6.1 six==1.11.0 typed-ast==1.1.0 wincertstore==0.2 [/code] となっており、ずれている、というわけです。 なので、githubのブランチをv4に変更して実行するのが正解。

    PIL がない

    [code lang=”cmd”] >python -i train_dcgan.py Traceback (most recent call last): File “train_dcgan.py”, line 14, in from visualize import out_generated_image File “D:\Works\Python\Envs\chainer\chainer\examples\dcgan\visualize.py”, line 6, in from PIL import Image ModuleNotFoundError: No module named ‘PIL [/code] なので、Pillowをインストール。 いつも、PILをインストールしようとするけど、そういう名前のパッケージはないのね。 [code lang=”cmd”] >python -m pip install Pillow Collecting Pillow Downloading https://files.pythonhosted.org/packages/bd/39/c76eaf781343162bdb1cf4854cb3bd5947a87ee44363e5acd6c48d69c4a1/Pillow-5.3.0-cp36-cp36m-win\_amd64.whl (1.6MB) 100% |████████████████████████████████| 1.6MB 975kB/s Installing collected packages: Pillow Successfully installed Pillow-5.3.0 [/code]
